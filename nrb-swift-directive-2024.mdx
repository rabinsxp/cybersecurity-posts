## Background

Following the 2017 NIC Asia SWIFT hack and subsequent incidents globally, Nepal Rastra Bank (NRB) issued updated directives requiring all licensed banks to implement enhanced SWIFT security controls. The directive, released in late 2023, mandates compliance with SWIFT's Customer Security Programme (CSP).

## Key Requirements

### 1. Mandatory Security Controls
NRB now requires all A and B class banks to:
- Implement multi-factor authentication for SWIFT terminals
- Maintain air-gapped or heavily segmented SWIFT infrastructure
- Conduct annual penetration testing by certified third parties

### 2. Incident Response Timeline
Banks must report any SWIFT-related security incident to NRB within **4 hours** of detection—a significantly shorter window than the previous 24-hour guideline.

### 3. Personnel Vetting
The directive introduces mandatory background checks for all personnel with access to SWIFT systems, including contractors.

## Industry Response

Most Class A banks have welcomed the directive, as many had already implemented these controls post-2017. However, smaller Class B institutions are struggling with the compliance costs, estimating implementation at NPR 10-50 million per institution.

## Analysis

This represents a significant step toward aligning Nepal's banking sector with international standards. However, the directive stops short of mandating **Security Operations Centers (SOCs)**—a gap that sophisticated attackers could still exploit.

## What This Means for Nepal

1. **Reduced Attack Surface**: Air-gapped SWIFT networks make remote attacks significantly harder.
2. **Faster Response**: The 4-hour reporting window enables coordinated responses.
3. **Talent Gap**: Nepal now desperately needs certified SWIFT security professionals.
